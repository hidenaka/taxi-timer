# タクシー乗務タイマー 設計ドキュメント

日付: 2026-04-24
作成者: hidenaka

## 1. 目的

タクシー乗務における休憩管理Webアプリ。

### 規制背景
- 連続で休憩せずに **6時間以上** 走行するとペナルティ
- 1日の合計休憩時間が **3時間（180分）** に達しない場合もペナルティ
- 「休憩」の成立条件: 停車時間が **11分以上**

### 解決する課題
- 連続走行時間のうっかり超過を防ぐ
- 合計休憩時間の進捗を可視化する
- 次にいつまでに休憩しなければならないかを即座に把握する

## 2. 機能要件

### 2.1 ストップウォッチ
- スタートボタンで計測開始
- 11分未満で中断する場合は「破棄」ボタンで計測リセット（記録しない）
- 11分経過後は「記録」ボタンが有効化される（11分未満では押せない）
- 「記録」を押すと、押した時刻と休憩時間（累積経過秒数）が保存され、ストップウォッチはリセット
- 11分経過後も記録ボタンを押すまで時計は回り続ける（実際の休憩長さを保存）

### 2.2 乗務開始時刻
- デフォルト値: `07:00`
- HTMLの `<input type="time">` でタップ編集可能
- localStorage に保持（次回起動時も前回の値を維持）
- 記録0件の場合、連続走行可能時刻の基準となる

### 2.3 主要表示項目
1. **今日の日付**: `2026/04/24 (金)` 形式
2. **乗務開始**: 編集可能な時刻入力
3. **ストップウォッチ**: `HH:MM:SS` 形式の大きな表示
4. **連続走行可能時刻**: 基準時刻 + 6時間 の実時刻（例: `13:00`）
   - 基準時刻 = 直近の記録時刻、または記録0件時は乗務開始時刻
5. **連続走行可能時間**: 連続走行可能時刻までの残り時間。**1秒ごとにリアルタイム更新**
6. **残り必要休憩**: `180分 - 合計休憩時間`。分と時間を並列表記（例: `45分 (0時間45分)`）
7. **合計休憩**: 記録済み休憩の累積（例: `135分 (2時間15分)`）
8. **記録履歴**: 新しい順に `時刻 / 休憩分数 / 次デッドライン時刻` を一覧表示

### 2.4 操作
- **スタート**: ストップウォッチ開始
- **記録**: 11分経過後のみ有効。記録保存＋ストップウォッチリセット
- **破棄**: 現在のストップウォッチを記録せずリセット
- **全リセット**: 記録履歴をすべてクリア（乗務開始時刻は保持）

## 3. 非機能要件

- **プラットフォーム**: iPad mini（メイン）、iPhone（サブ）Safari
- **データ永続化**: localStorage。1日は最低限保持されればよい。自動リセットなし（手動リセットのみ）
- **全画面表示**: 不要（通常のSafariタブで使用）
- **オフライン動作**: 一度ロードすれば通信不要（ページ再読み込みは要通信）
- **保守性**: 単一HTMLファイルで完結させ、依存ライブラリ・ビルドツールなし

## 5. データモデル

### 5.1 localStorage スキーマ
キー: `taxi-timer-v1`

```json
{
  "shiftStart": "07:00",
  "records": [
    {
      "recordedAt": "2026-04-24T08:42:15+09:00",
      "durationSec": 1080
    }
  ]
}
```

### 5.2 派生値の計算
- **基準時刻**: `records[last].recordedAt` 、または records 空なら 今日の `shiftStart`
- **連続走行可能時刻**: `基準時刻 + 6時間`
- **連続走行可能時間**: `連続走行可能時刻 - 現在時刻`（マイナスなら `超過: Xh Ym`）
- **合計休憩（分）**: `sum(records[].durationSec) / 60` を切り捨て
- **残り必要休憩（分）**: `max(0, 180 - 合計休憩)`

## 6. UI 仕様

### 6.1 レイアウト
```
┌─────────────────────────────────┐
│  タクシー乗務タイマー              │
│  2026/04/24 (金)                 │
├─────────────────────────────────┤
│ 乗務開始: [ 07:00 ]              │
├─────────────────────────────────┤
│   ⏱  00:14:32                   │
│   [ スタート ] [ 記録 ] [ 破棄 ]  │
├─────────────────────────────────┤
│ 連続走行可能時刻: 13:00          │
│ 連続走行可能時間: 2時間15分      │
│ 残り必要休憩: 180分 (3時間)      │
│ 合計休憩: 0分 (0時間)           │
├─────────────────────────────────┤
│ 記録履歴（0件）                  │
├─────────────────────────────────┤
│          [ 全リセット ]          │
└─────────────────────────────────┘
```

### 6.2 ボタン状態
| 状態 | スタート | 記録 | 破棄 |
|---|---|---|---|
| 停止中 | 有効 | 無効 | 無効 |
| 計測中・11分未満 | 無効 | 無効 | 有効 |
| 計測中・11分以上 | 無効 | 有効 | 有効 |

### 6.3 視覚的な注意喚起
- 連続走行可能時間が **残り60分以下** → 表示をオレンジ色に
- 連続走行可能時間が **残り30分以下** → 表示を赤色に
- 連続走行可能時間が **超過**（負の値）→ 赤背景 + 「超過」表示

### 6.4 タッチ最適化
- ボタンは最小 44px（iOS推奨のタッチターゲット）
- フォントは iPad mini で読みやすいサイズ（ストップウォッチ部分は 48px 以上）
